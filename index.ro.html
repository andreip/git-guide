crează un nou depozit git

crează un director nou, deschide-l și realizează comanda
git init
pentru a crea un nou depozit git.

investighează un depozit git

crează o copie de lucru a unui depozit local prin rularea comenzii
git clone /cale/către/depozit
când folosiți un server la distanță, comanda va fi
git clone username@host:/cale/către/depozit

pași de lucru

depozitul local este format din trei "arbori" menținuți de git. primul este Directorul de lucru care conține fișierele efective. al doilea se numește Index și are rolul unei zone intermediare. ultimul se numește HEAD și arată spre ultimul commit făcut.

adaugă & comite

Puteți propune schimbări (adăugați-le în Index) folosind
git add <nume_fișier>
git add *
Acesta e primul pas în lucratul cu git. Pentru a comite cu adevărat aceste schimbări folosiți
git commit -m "Mesaj despre cele comise"
Acum schimbările sunt înregistrate în HEAD, dar încă nu în depozitul la distanță.

trimiterea schimbărilor

Schimbările sunt acum în HEAD-ul din spațiul local de lucru. Pentru a trimite aceste schimbări în depozitul la distanță, executați
git push origin master
Înlocuiți master cu orice branch (ramură din arbore) în care doriți să trimiteți schimbările.

Dacă nu ați clonat un depozit existent și doriți să conectați / legați depozitul de un server la distanță, executați
git remote add origin <server>
Acum puteți trimite schimbările către serverul la distantă selectat

lucrul pe ramuri (branch)

Ramurile sunt folosite pentru dezvoltarea unor funcționalități disjuncte / diferite. Ramura master este cea "implicită" la crearea unui nou depozit. Folosiți alte ramuri pentru dezvoltare și apoi îmbinați-le (prin merge) în master când ați terminat funcționalitatea.

crează o nouă ramură numită "feature_x" și mută-te pe ea prin
git checkout -b feature_x
mută-te înapoi pe ramura master
git checkout master
și șterge ramura nou creată
git branch -d feature_x
o ramură nu e disponibilă altora până nu adăugați acea ramură în depozitul la distanță
git push origin <ramură>

actualizare & îmbinare

pentru a actualiza depozitul local la ultimul lucru comis, executați
git pull
în directorul de lucru pentru a lua (fetch) și a îmbina (merge) schimbările de la distanță.
pentru a îmbina o altă ramură în ramura curentă (ex. master), folosiți
git merge <ramură>
în ambele cazuri git încearcă să îmbine automat schimbările. Din nefericire, asta nu se poate întotdeauna, uneori rezultând conflicte. Este responsabilitatea proprie să îmbinați acele conflicte manual prin editarea acelor fișiere arătate de git. După realizarea schimbărilor, trebuie să le marcați ca îmbinate cu
git add <nume_fișier>
înainte să îmbinați schimbările, puteți să le vedeți prin comanda
git diff <ramură_sursă> <ramură_țintă>

etichete

e recomandat să creați etichete (tag-uri) pentru release-uri de software, concept existent și în SVN. Puteți crea o nouă etichetă numită 1.0.0 prin
git tag 1.0.0 1b2e1d63ff
codul 1b2e1d63ff corespunde primelor 10 caractere din identificatorul înregistrării comise pe care doriți s-o etichetați. Puteți obține identificatorul din
git log
puteți folosi chiar și mai puține caractere din identificator, atâta timp cât este unic.

înlocuitul schimbărilor locale

În caz că ați făcut ceva greșit (sigur nu se va întâmpla ;) puteți repara cu comanda
git checkout -- <nume_fișier>
comanda înlocuiește schimbările din arborele de lucru (working tree) cu ultima înregistrare comisă din HEAD. Schimbările deja adăugate în index, precum și fișierele nou create, nu vor fi schimbate.

Dacă în schimb doriți să aruncați toate schimbările făcute local, puteți lua ultimele înregistrări de la server și să setați ramura locală master să arate spre ele astfel
git fetch origin
git reset --hard origin/master

sugestii utile

interfață grafică ce vine cu git
gitk
mesaje de ieșire colorate
git config color.ui true
mesaje de log (jurnale) pe o singură linie
git config format.pretty oneline
adăugare interactivă în index
git add -i
